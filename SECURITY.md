# Security Policy

## Supported scope

Security issues are relevant for:

- authentication and authorization
- token handling and session flows
- sensitive user data exposure
- abuse-reporting and travel-partner safety flows
- admin access controls
- dependency or infrastructure vulnerabilities that materially affect deployment security

## Reporting a vulnerability

Do not open a public GitHub issue for a suspected security vulnerability.

Instead:

1. Email the maintainer at `tanikkumar4@gmail.com`
2. Use the subject line `GHUMLE SECURITY REPORT`
3. Include reproduction steps, impact, affected files or routes, and any mitigation ideas

You should receive an acknowledgment within 5 business days.

## Disclosure expectations

- Give maintainers reasonable time to investigate and patch
- Avoid public disclosure before a fix or coordinated response
- Provide enough detail to reproduce and validate the issue
